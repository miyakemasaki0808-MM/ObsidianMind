# セクション単位AIチャット機能 実装報告書

**日付:** 2026-07-17
**プロジェクト:** Obsidian Mind
**対象ブランチ:** feature/Additional_Function

---

## 1. 背景・目的

長いノートに対して「今見ている部分だけ」を要約・質問したいという要望を受け、ノート画面に浮遊する吹き出しボタンを配置し、スクロールで**今見えているセクション（見出し＋その配下）**を自動追従して対象にする機能を実装した。

タップすると、そのセクションだけを文脈にした**要約**と**質問候補**をローカルLLM（Gemini Nano／`AiClient`）で生成する。全文でなく該当セクションのみを渡すため、長いノートでもコンテキスト長に収まりやすく、回答の的確さも上がる。

デザインは複数回のイテレーションを経て、以下の仕様に確定した。

- 入口: 画面に浮かぶ半透明・立体的な吹き出しボタン（ドラッグで移動可）。白は使わずアプリのアクセント色（Indigo→Coral）でノートと区別しやすくする。
- 対象決定: 押した時点で画面上端に見えている見出し＋配下（次の同レベル以下の見出し直前まで）。見出しの無いノートはノート全体にフォールバック。
- タップ後: 自由記述欄は設けず、ボトムシートに**要約（上）／質問候補タップ（下）**を配置。
- 参照範囲: 現在のノート（当該セクション）のみ。会話履歴はシートを開いている間だけ保持（揮発）。

## 2. 実装内容

### 2-1. セクション分割ロジック（新規）

本文は既存の `parseMarkdownBlocks()` で見出し・段落・リスト等のブロック列に分解済み。これを流用し、見出しごとに「その見出し〜次の同レベル以下の見出し直前」までをセクションとして再構成する。

| 要素 | 内容 |
|------|------|
| `NoteSection` | セクションのタイトル・レベル・LLM入力用テキスト |
| `NoteSectionModel` | `sectionForBlockIndex(index)` — ブロックindexから直近の見出しセクションを解決 |
| `buildNoteSectionModel(content)` | 本文からセクションモデルを構築 |
| `blocksToMarkdown(blocks)` | パース済みブロックをLLM入力用Markdownに再構成 |

### 2-2. 可視位置の取得

本文表示（`MarkdownNoteContent`）を `Column(verticalScroll)` から **`LazyColumn`** に変更し、`listState` を外部から受け取れるようにした。`listState.firstVisibleItemIndex` が「今上端に見えているブロック」に一致するため、これを `sectionForBlockIndex` に渡してリアルタイムに対象セクションを解決する。既存の呼び出し（補記メモ画面等）はデフォルト引数により無改修で動作する。

### 2-3. 浮遊吹き出し（FAB）

`NoteReaderTab` にオーバーレイとして配置。

- 半透明のIndigo→Coralグラデーション＋白リムなしの色付きガラス調
- `shadow(18dp)` による立体感、上部にAqua半透明のハイライト（艶）
- ドラッグで自由に移動可能（`detectDragGestures`）
- タップで直接ボトムシートを開く（中間メニューは廃止）
- ラベルに現在の対象セクション名を表示し、スクロールに追従して切り替わる

### 2-4. AIチャット状態・プロンプト

`NoteViewModel` に `SectionChatState` を追加し、`openSection(section)` で要約と質問候補をまとめて生成する単一の入口に統合した。

| 追加関数（`PromptBuilder`） | 役割 |
|---|---|
| `buildSectionSummaryPrompt` | セクションのみを2〜4文で要約 |
| `buildSectionSuggestionsPrompt` | セクションについての質問候補を最大3件生成 |
| `buildSectionChatPrompt` | セクションの内容のみを根拠に回答（範囲外は「記載なし」と回答させる） |

### 2-5. ボトムシートUI（新規）

`SectionChatSheet` を新規実装。自由記述欄は持たず、以下の構成に統一。

1. スコープ表示（📌 セクション名）
2. 📝 要約（上部）
3. 💬 質問候補（下部・タップ式、押すとその場でQ&Aログに追加）

## 4. デザインの反復と決定事項

開発途中でユーザーからのフィードバックを受け、以下のとおり方向修正を行った。

| 段階 | 内容 | 対応 |
|------|------|------|
| 初期案 | 見出しごとに個別チップを配置する案 | 「楽天のような浮遊ボタン」イメージと異なるため不採用 |
| 修正1 | 画面右下に浮遊するドラッグ可能な吹き出しに変更 | 採用 |
| 修正2 | 吹き出しタップ後、自由記述欄・中間メニューは不要 | 自由記述欄を撤去し、タップで直接シートを開く形に変更 |
| 修正3 | シートのレイアウトが手狭 | 要約（上）／質問候補タップ（下）の縦構成に変更、シート高さ・余白を拡大 |
| 修正4 | 吹き出しが白基調でノートに同化しにくい | 白を排し、Indigo→Coralの半透明グラデーションに変更 |
| 修正5 | シートタイトルの「（配下を含む）」表記が不要 | 削除し、セクション名のみの表示に変更 |

## 6. 今後の拡張余地

- 現状は「現在のノートのセクション」のみが参照範囲。将来、関連ノートを含めた横断チャットへの拡張が検討候補。
- 会話履歴は揮発（シートを閉じると破棄）。永続化のニーズが出た場合は保存先設計が別途必要。
