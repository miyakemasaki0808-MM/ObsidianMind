# 設計思想 — AI生成のバックグラウンドUX

**対象領域:** AI生成（要約・Q&A・補記）の待ち時間と結果通知
**初版:** 2026-07-19〜20（PR #22, #23）

---

## 解決したかった問題

オンデバイスのGemini Nanoは生成に数十秒かかることがあり、さらに `AiClient` のMutexで直列化されているため順番待ちも発生する。従来は生成開始と同時に待機画面へ遷移し、ユーザーはその間ノートを読めなかった。「生成はAIの都合、読書はユーザーの都合。前者が後者をブロックしてはいけない」が本領域の基本思想。

## PR #22: AI要約中もノート閲覧を継続

要約待ちでノート本文の閲覧がブロックされる問題を解消し、「生成を待たせながら読み続ける」方向性の最初の一歩とした。

## PR #23: Q&A・補記の待機画面を廃止

生成開始後は待機画面へ遷移せず、ノートタブに戻して読書を継続させる。完了・エラーは次の3層で知らせる。

1. **Snackbar** — 開始「作成中…」／完了「保存しました＋見る」／失敗「作成できませんでした＋詳細」
2. **AIタブのバッジ** — 未確認の重要度順に1状態だけ表示（エラー > 未確認完了 > 生成中）。タブの✨の意味は変えず、右上の小バッジのみで伝える
3. **AIタブ内ボタンのラベル多態** — 状態に応じて「作る／作成中…／✓開く／!エラーを確認／↻再試行」と役割が変わる

### 状態設計: isViewed（未確認管理）

Success/Errorに `isViewed` フラグを持たせ、「結果はあるがユーザーがまだ見ていない」を表現する。通知・バッジは未確認のものだけを対象にし、結果を開いた時点で確認済みにする。結果自体は同じノート内では保持し続ける。

### 競合ガード: requestId + isCurrent

ノート切替後に旧ノートの生成結果が新しい画面へ混入するのを防ぐため、要求ごとに `requestId` を採番し、suspend地点の後で `isCurrent(requestId)` を確認してから状態を更新する。Jobキャンセルだけに頼らないのは、キャンセルが届く前にすり抜ける完了通知（モデルDLコールバック等）があるため。

### 例外方針

`CancellationException` は握りつぶさず再throwする。広い `catch (e: Exception)` でキャンセルをエラー表示に変換すると、ノート切替のたびに偽のエラーが出る。

## Controller構成

Q&Aは `NoteViewModel` ベタ書きから `QuizController` に切り出し、`AnnotationController` と相似形にした。**共通化は意図的に保留**している。2件では抽象の形を決めるには早く、3件目のバックグラウンドAI機能が来た時点で共通パターンを抽出する判断とした。

## 追記

### 2026-07-20 — 画面回転でSnackbarが再表示される問題（レビュー対応）

`remember { SnackbarHostState() }` はActivity再生成で初期化され、状態から通知を導出する `LaunchedEffect` が再実行されて同じSnackbarが再表示されていた。「Snackbarは一度きりのイベント、生成状態は持続する状態」という性質の違いが原因。

対処は表示済みイベントキーを `rememberSaveable` に記録して二重表示を抑止。状態がIdleに戻ったらキーもリセットし、同じノートの再生成では再び通知できるようにする。**割り切り: 未確認のSuccessも回転後は再表示しない（バッジが未確認を伝え続けるため通知の再表示は不要と判断）。**

あわせてイベントキーの組み立てを `toEventKey()` 拡張関数へ切り出してテスト可能にした。バッジ優先順位（`resolveAiTabBadgeState`）も同様に純関数化されており、**UI分岐は関数に切り出せばそのままテストになる**という定石に従っている。

### 2026-07-20 — Q&Aの起点移動に伴う通知3層の再編

Q&Aの入口が吹き出しシートへ移動した（[section_ai_chat](section_ai_chat.md) 参照）ことで、3層通知のうちQ&Aに残るのは **Snackbar（開始／完了＋始める／失敗＋詳細）** と **シート内ボタンのラベル多態** になり、**AIタブバッジの対象は補記メモのみ**になった。AIタブから開けない機能のバッジをAIタブに出すと誘導先が存在しないため。`resolveAiTabBadgeState` は補記のみを引数に取る形へ単純化した。
