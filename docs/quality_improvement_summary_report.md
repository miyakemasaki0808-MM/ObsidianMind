## 5. PR #19 — NoteViewModel 分割（5コミット）

**R2** を「単一ViewModel＋機能Controller」方式で解消。公開APIと `uiState` を維持したままUI層の変更ゼロで、906行→348行（-62%）に分割した。

| ファイル | 責務 |
|---|------|
| `NoteUiState.kt` | 状態定義12個（純移動） |
| `SectionChatController.kt` / `AnnotationController.kt` / `SearchController.kt` | 機能単位のController（scope・状態Flow注入） |
| `QuizResponseParser.kt` / `AnnotationComposer.kt` | Android非依存の純関数（ユニットテスト可能に） |

結合点は `cancelJobs()`（ノート切替）／`onVaultChanged()`（Vault切替）という明示契約に変換。以後のAI機能追加は「Controller 1ファイル＋状態1フィールド＋契約2箇所への登録」の定型となる。実機検証済み。
